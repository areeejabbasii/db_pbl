================================================================================
RELATIONSHIP DEFINITIONS
================================================================================

RELATIONSHIP 1: Categories → Books
─────────────────────────────────

Type: 1:M (One-to-Many)
Participation: 
  - Books: TOTAL (every book must belong to a category)
  - Categories: PARTIAL (a category may have no books)
Cardinality: 1 (Categories) → M (Books)
Action: ON DELETE CASCADE (delete category removes all its books)
SQL: FOREIGN KEY (CategoryID) REFERENCES Categories(CategoryID)

RELATIONSHIP 2: Authors ←→ Books (M:M)
──────────────────────────────────────

Type: M:M (Many-to-Many)
Via: Book_Authors (Associative Entity)
Participation:
  - Authors: TOTAL (every author must write at least one book)
  - Books: TOTAL (every book must have at least one author)
Cardinality: M (Authors) ←→ M (Books)
Additional Attribute: AuthorSequence (indicates author order)
Unique Constraint: UNIQUE(BookID, AuthorID) - no duplicate combinations
Action: ON DELETE CASCADE

RELATIONSHIP 3: Members → Borrowing
───────────────────────────────────

Type: 1:M (One-to-Many)
Participation:
  - Borrowing: TOTAL (every borrowing record belongs to a member)
  - Members: PARTIAL (a member may not have any borrowing records)
Cardinality: 1 (Members) → M (Borrowing)
Action: ON DELETE CASCADE
SQL: FOREIGN KEY (MemberID) REFERENCES Members(MemberID)

RELATIONSHIP 4: Books → Borrowing
──────────────────────────────────

Type: 1:M (One-to-Many)
Participation:
  - Borrowing: TOTAL (every borrowing references a book)
  - Books: PARTIAL (a book may never be borrowed)
Cardinality: 1 (Books) → M (Borrowing)
Action: ON DELETE CASCADE
SQL: FOREIGN KEY (BookID) REFERENCES Books(BookID)

RELATIONSHIP 5: Borrowing → Fines
──────────────────────────────────

Type: 1:M (One-to-Many)
Participation:
  - Fines: TOTAL (every fine relates to a borrowing)
  - Borrowing: PARTIAL (not all borrowings result in fines)
Cardinality: 1 (Borrowing) → M (Fines)
Action: ON DELETE NO ACTION (preserve fine history)
SQL: FOREIGN KEY (BorrowingID) REFERENCES Borrowing(BorrowingID)

RELATIONSHIP 6: Members → Fines
────────────────────────────────

Type: 1:M (One-to-Many)
Participation:
  - Fines: TOTAL (every fine belongs to a member)
  - Members: PARTIAL (a member may have no fines)
Cardinality: 1 (Members) → M (Fines)
Action: ON DELETE NO ACTION
SQL: FOREIGN KEY (MemberID) REFERENCES Members(MemberID)

RELATIONSHIP 7: Members → MemberStatusAudit
─────────────────────────────────────────────

Type: 1:M (One-to-Many)
Participation: TOTAL (every audit record belongs to a member)
Cardinality: 1 (Members) → M (MemberStatusAudit)
Action: ON DELETE CASCADE
SQL: FOREIGN KEY (MemberID) REFERENCES Members(MemberID)

CARDINALITY NOTATION LEGEND
================================================================================

1:1 - One-to-One
     Each entity in set A relates to at most one entity in set B
     Each entity in set B relates to at most one entity in set A

1:M - One-to-Many
     Each entity in set A relates to many entities in set B
     Each entity in set B relates to at most one entity in set A

M:M - Many-to-Many
     Each entity in set A relates to many entities in set B
     Each entity in set B relates to many entities in set A
     Requires associative entity

─── - Partial Participation (optional)
     Not all entities in set must participate in relationship

─── - Total Participation (mandatory)
     All entities in set must participate in relationship
